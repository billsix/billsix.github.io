GUI Programming¶

Introduction to GUI Programming¶

The purpose of this appendix is not to teach you how to do Graphical User Interfaces. It is simply meant to show how writing graphical applications is the same as writing other applications, just using an additional library to handle the graphical parts. As a programmer you need to get used to learning new libraries. Most of your time will be spent passing data from one library to another.

The GNOME Libraries¶

The GNOME projects is one of several projects to provide a complete desktop to Linux users. The GNOME project includes a panel to hold application launchers and mini-applications called applets, several standard applications to do things such as file management, session management, and configuration, and an API for creating applications which fit in with the way the rest of the system works.

One thing to notice about the GNOME libraries is that they constantly create and give you pointers to large data structures, but you never need to know how they are laid out in memory. All manipulation of the GUI data structures are done entirely through function calls. This is a characteristic of good library design. Libraries change from version to version, and so does the data that each data structure holds. If you had to access and manipulate that data yourself, then when the library is updated you would have to modify your programs to work with the new library, or at least recompile them. When you access the data through functions, the functions take care of knowing where in the structure each piece of data is. The pointers you receive from the library are opaque - you don’t need to know specifically what the structure they are pointing to looks like, you only need to know the functions that will properly manipulate it. When designing libraries, even for use within only one program, this is a good practice to keep in mind.

This chapter will not go into details about how GNOME works. If you would like to know more, visit the GNOME developer web site at http://developer.gnome.org/. This site contains tutorials, mailing lists, API documentation, and everything else you need to start programming in the GNOME environment.

A Simple GNOME Program in Several Languages¶

This program will simply show a Window that has a button to quit the application. When that button is clicked it will ask you if you are sure, and if you click yes it will close the application. To run this program, type in the following as gnome-example.s:

src/gnome-example.s¶
  1#PURPOSE:  This program is meant to be an example
  2#          of what GUI programs look like written
  3#          with the GNOME libraries
  4#
  5#INPUT:    The user can only click on the "Quit"
  6#          button or close the window
  7#
  8#OUTPUT:   The application will close
  9#
 10#PROCESS:  If the user clicks on the "Quit" button,
 11#          the program will display a dialog asking
 12#          if they are sure.  If they click Yes, it
 13#          will close the application.  Otherwise
 14#          it will continue running
 15#          
 16
 17.section .data
 18

 20#                      header files for the C language
 21#                      and converted into their assembly
 22#                      equivalents
 23
 24#GNOME Button Names
 25GNOME_STOCK_BUTTON_YES:
 26.ascii "Button_Yes\0"
 27GNOME_STOCK_BUTTON_NO:
 28.ascii "Button_No\0"
 29
 30#Gnome MessageBox Types
 31GNOME_MESSAGE_BOX_QUESTION:
 32.ascii "question\0"
 33
 34#Standard definition of NULL
 35.equ NULL, 0
 36
 37#GNOME signal definitions
 38signal_destroy:
 39.ascii "destroy\0"
 40signal_delete_event:
 41.ascii "delete_event\0"
 42signal_clicked:
 43.ascii "clicked\0"
 44
 45###Application-specific definitions
 46
 47#Application information
 48app_id:
 49.ascii "gnome-example\0"
 50app_version:
 51.ascii "1.000\0"
 52app_title:
 53.ascii "Gnome Example Program\0"
 54
 55#Text for Buttons and windows
 56button_quit_text:
 57.ascii "I Want to Quit the GNOME Example Program\0"
 58quit_question:
 59.ascii "Are you sure you want to quit?\0"
 60
 61
 62.section .bss
 63
 64#Variables to save the created widgets in
 65.equ WORD_SIZE, 4
 66.lcomm appPtr, WORD_SIZE
 67.lcomm btnQuit, WORD_SIZE
 68
 69.section .text
 70
 71.globl main
 72.type main,@function
 73main:
 74	pushl %ebp
 75	movl  %esp, %ebp
 76
 77	#Initialize GNOME libraries
 78	pushl 12(%ebp)     #argv
 79	pushl 8(%ebp)      #argc
 80	pushl $app_version 
 81	pushl $app_id      
 82	call  gnome_init
 83	addl  $16, %esp    #recover the stack
 84
 85	#Create new application window
 86	pushl $app_title    #Window title
 87	pushl $app_id       #Application ID
 88	call  gnome_app_new
 89	addl  $8, %esp      #recover the stack
 90	movl  %eax, appPtr  #save the window pointer
 91
 92	#Create new button
 93	pushl $button_quit_text #button text
 94	call  gtk_button_new_with_label
 95	addl  $4, %esp          #recover the stack
 96	movl  %eax, btnQuit     #save the button pointer
 97
 98	#Make the button show up inside the application window
 99	pushl btnQuit
100	pushl appPtr
101	call  gnome_app_set_contents
102	addl  $8, %esp
103
104	#Makes the button show up (only after it's window
105	#shows up, though)
106	pushl btnQuit
107	call  gtk_widget_show
108	addl  $4, %esp
109
110	#Makes the application window show up
111	pushl appPtr
112	call  gtk_widget_show
113	addl  $4, %esp
114
115	#Have GNOME call our delete_handler function
116	#whenever a "delete" event occurs
117	pushl $NULL            #extra data to pass to our
118	                       #function (we don't use any)
119	pushl $delete_handler  #function address to call
120	pushl $signal_delete_event #name of the signal
121	pushl appPtr           #widget to listen for events on
122	call  gtk_signal_connect
123	addl  $16, %esp        #recover stack
124
125	#Have GNOME call our destroy_handler function
126	#whenever a "destroy" event occurs
127	pushl $NULL            #extra data to pass to our
128	                       #function (we don't use any)
129	pushl $destroy_handler #function address to call
130	pushl $signal_destroy  #name of the signal
131	pushl appPtr           #widget to listen for events on
132	call  gtk_signal_connect
133	addl  $16, %esp        #recover stack
134
135	#Have GNOME call our click_handler function
136	#whenever a "click" event occurs.  Note that
137	#the previous signals were listening on the
138	#application window, while this one is only
139	#listening on the button
140	pushl $NULL
141	pushl $click_handler
142	pushl $signal_clicked
143	pushl btnQuit
144	call  gtk_signal_connect
145	addl  $16, %esp
146
147	#Transfer control to GNOME.  Everything that
148	#happens from here out is in reaction to user
149	#events, which call signal handlers.  This main
150	#function just sets up the main window and connects
151	#signal handlers, and the signal handlers take 
152	#care of the rest
153	call  gtk_main
154
155	#After the program is finished, leave
156	movl  $0, %eax
157	leave
158	ret
159
160#A "destroy" event happens when the widget is being
161#removed.  In this case, when the application window
162#is being removed, we simply want the event loop to
163#quit
164destroy_handler:
165	pushl %ebp
166	movl  %esp, %ebp
167
168	#This causes gtk to exit it's event loop
169	#as soon as it can.
170	call  gtk_main_quit
171
172	movl  $0, %eax
173	leave
174	ret
175
176#A "delete" event happens when the application window 
177#gets clicked in the "x" that you normally use to 
178#close a window
179delete_handler:
180	movl  $1, %eax
181	ret
182
183#A "click" event happens when the widget gets clicked
184click_handler:
185	pushl %ebp
186	movl  %esp, %ebp
187
188	#Create the "Are you sure" dialog
189	pushl $NULL                       #End of buttons
190	pushl $GNOME_STOCK_BUTTON_NO      #Button 1
191	pushl $GNOME_STOCK_BUTTON_YES     #Button 0
192	pushl $GNOME_MESSAGE_BOX_QUESTION #Dialog type
193	pushl $quit_question              #Dialog mesasge
194	call  gnome_message_box_new
195	addl  $16, %esp                   #recover stack
196
197	#%eax now holds the pointer to the dialog window
198
199	#Setting Modal to 1 prevents any other user
200	#interaction while the dialog is being shown
201	pushl $1
202	pushl %eax
203	call  gtk_window_set_modal
204	popl  %eax
205	addl  $4, %esp
206
207	#Now we show the dialog	
208	pushl %eax
209	call  gtk_widget_show
210	popl  %eax
211
212	#This sets up all the necessary signal handlers
213	#in order to just show the dialog, close it when
214	#one of the buttons is clicked, and return the
215	#number of the button that the user clicked on.
216	#The button number is based on the order the buttons
217	#were pushed on in the gnome_message_box_new function
218	pushl %eax	
219	call  gnome_dialog_run_and_close
220	addl  $4, %esp
221
222	#Button 0 is the Yes button.  If this is the
223	#button they clicked on, tell GNOME to quit
224	#it's event loop.  Otherwise, do nothing
225	cmpl  $0, %eax
226	jne   click_handler_end
227
228	call  gtk_main_quit
229
230click_handler_end:
231	leave
232	ret

To build this application, execute the following commands:

as gnome-example.s -o gnome-example.o
gcc gnome-example.o `gnome-config --libs gnomeui` \
    -o gnome-example

Then type in ./gnome-example to run it.

This program, like most GUI programs, makes heavy use of passing pointers to functions as parameters. In this program you create widgets with the GNOME functions and then you set up functions to be called when certain events happen. These functions are called callback functions. All of the event processing is handled by the function gtk_main, so you don’t have to worry about how the events are being processed. All you have to do is have callbacks set up to wait for them.

Here is a short description of all of the GNOME functions that were used in this program:

gnome_init

Takes the command-line arguments, argument count, application id, and application version and initializes the GNOME libraries.

gnome_app_new

Creates a new application window, and returns a pointer to it. Takes the application id and the window title as arguments.

gtk_button_new_with_label

Creates a new button and returns a pointer to it. Takes one argument - the text that is in the button.

gnome_app_set_contents

This takes a pointer to the gnome application window and whatever widget you want (a button in this case) and makes the widget be the contents of the application window

gtk_widget_show

This must be called on every widget created (application window, buttons, text entry boxes, etc) in order for them to be visible. However, in order for a given widget to be visible, all of its parents must be visible as well.

gtk_signal_connect

This is the function that connects widgets and their signal handling callback functions. This function takes the widget pointer, the name of the signal, the callback function, and an extra data pointer. After this function is called, any time the given event is triggered, the callback will be called with the widget that produced the signal and the extra data pointer. In this application, we don’t use the extra data pointer, so we just set it to NULL, which is 0.

gtk_main

This function causes GNOME to enter into its main loop. To make application programming easier, GNOME handles the main loop of the program for us. GNOME will check for events and call the appropriate callback functions when they occur. This function will continue to process events until gtk_main_quit is called by a signal handler.

gtk_main_quit

This function causes GNOME to exit its main loop at the earliest opportunity.

gnome_message_box_new

This function creates a dialog window containing a question and response buttons. It takes as parameters the message to display, the type of message it is (warning, question, etc), and a list of buttons to display. The final parameter should be NULL to indicate that there are no more buttons to display.

gtk_window_set_modal

This function makes the given window a modal window. In GUI programming, a modal window is one that prevents event processing in other windows until that window is closed. This is often used with Dialog windows.

gnome_dialog_run_and_close

This function takes a dialog pointer (the pointer returned by gnome_message_box_new can be used here) and will set up all of the appropriate signal handlers so that it will run until a button is pressed. At that time it will close the dialog and return to you which button was pressed. The button number refers to the order in which the buttons were set up in gnome_message_box_new.

The following is the same program written in the C language. Type it in as gnome-example-c.c:

src/gnome-example-c.c¶
 1#include <gtk/gtk.h>
 2
 3#define MY_APP_TITLE "Gnome Example Program"
 4#define MY_APP_ID "org.example.gnome_example"
 5#define MY_BUTTON_TEXT "I Want to Quit the Example Program"
 6#define MY_QUIT_QUESTION "Are you sure you want to quit?"
 7
 8/* Dialog response handler */
 9static void dialog_response_cb(GtkDialog *dialog, gint response_id,
10                               gpointer user_data) {
11  GApplication *app = G_APPLICATION(user_data);
12  if (response_id == GTK_RESPONSE_YES) {
13    g_application_quit(app);
14  }
15  gtk_window_destroy(GTK_WINDOW(dialog));
16}
17
18/* Show modal confirmation dialog */
19static void show_quit_dialog(GtkWindow *parent, GApplication *app) {
20  GtkWidget *dialog = gtk_dialog_new_with_buttons(
21      "Confirm", parent, GTK_DIALOG_MODAL, "_Yes", GTK_RESPONSE_YES, "_No",
22      GTK_RESPONSE_NO, NULL);
23
24  GtkWidget *label = gtk_label_new(MY_QUIT_QUESTION);
25  gtk_label_set_wrap(GTK_LABEL(label), TRUE);
26
27  GtkWidget *content_area = gtk_dialog_get_content_area(GTK_DIALOG(dialog));
28  gtk_box_append(GTK_BOX(content_area), label);
29
30  g_signal_connect(dialog, "response", G_CALLBACK(dialog_response_cb), app);
31
32  gtk_window_present(GTK_WINDOW(dialog));
33}
34
35/* Button clicked */
36static void on_button_clicked(GtkButton *button, gpointer user_data) {
37  GApplication *app = G_APPLICATION(user_data);
38  GtkWindow *parent = GTK_WINDOW(gtk_widget_get_root(GTK_WIDGET(button)));
39  show_quit_dialog(parent, app);
40}
41
42/* Window close request */
43static gboolean on_close_request(GtkWindow *window, gpointer user_data) {
44  GApplication *app = G_APPLICATION(user_data);
45  show_quit_dialog(window, app);
46  return TRUE;  // prevent automatic closing
47}
48
49/* Application activate */
50static void on_activate(GApplication *app, gpointer user_data) {
51  GtkWidget *window = gtk_application_window_new(GTK_APPLICATION(app));
52  gtk_window_set_title(GTK_WINDOW(window), MY_APP_TITLE);
53  gtk_window_set_default_size(GTK_WINDOW(window), 400, 100);
54
55  GtkWidget *button = gtk_button_new_with_label(MY_BUTTON_TEXT);
56  gtk_window_set_child(GTK_WINDOW(window), button);
57
58  g_signal_connect(button, "clicked", G_CALLBACK(on_button_clicked), app);
59  g_signal_connect(window, "close-request", G_CALLBACK(on_close_request), app);
60
61  gtk_window_present(GTK_WINDOW(window));
62}
63
64int main(int argc, char **argv) {
65  GtkApplication *app =
66      gtk_application_new(MY_APP_ID, G_APPLICATION_FLAGS_NONE);
67  g_signal_connect(app, "activate", G_CALLBACK(on_activate), NULL);
68
69  int status = g_application_run(G_APPLICATION(app), argc, argv);
70  g_object_unref(app);
71  return status;
72}

To compile it, type

gcc gnome-example-c.c `gnome-config --cflags \
    --libs gnomeui` -o gnome-example-c

Run it by typing ./gnome-example-c.

Finally, we have a version in Python. Type it in as gnome-example.py:

src/gnome-example.py¶
 1#PURPOSE:  This program is meant to be an example
 2#          of what GUI programs look like written
 3#          with the GNOME libraries
 4#
 5
 6#Import GNOME libraries
 7import gtk
 8import gnome.ui
 9
10####DEFINE CALLBACK FUNCTIONS FIRST####
11
12#In Python, functions have to be defined before 
13#they are used, so we have to define our callback 
14#functions first.
15
16def destroy_handler(event):
17	gtk.mainquit()
18	return 0
19
20def delete_handler(window, event):
21	return 0
22
23def click_handler(event):
24	#Create the "Are you sure" dialog
25	msgbox = gnome.ui.GnomeMessageBox(
26		"Are you sure you want to quit?", 
27		gnome.ui.MESSAGE_BOX_QUESTION, 
28		gnome.ui.STOCK_BUTTON_YES, 
29		gnome.ui.STOCK_BUTTON_NO)
30	msgbox.set_modal(1)
31	msgbox.show()
32
33	result = msgbox.run_and_close()
34
35	#Button 0 is the Yes button.  If this is the
36	#button they clicked on, tell GNOME to quit
37	#it's event loop.  Otherwise, do nothing
38	if (result == 0):
39		gtk.mainquit()	
40
41	return 0
42
43####MAIN PROGRAM####
44
45#Create new application window
46myapp = gnome.ui.GnomeApp(
47	"gnome-example", "Gnome Example Program")
48
49#Create new button
50mybutton = gtk.GtkButton(
51	"I Want to Quit the GNOME Example program")
52myapp.set_contents(mybutton)
53
54#Makes the button show up 
55mybutton.show()
56
57#Makes the application window show up
58myapp.show()
59
60#Connect signal handlers
61myapp.connect("delete_event", delete_handler)
62myapp.connect("destroy", destroy_handler)
63mybutton.connect("clicked", click_handler)
64
65#Transfer control to GNOME
66gtk.mainloop()
67

To run it type python gnome-example.py.

GUI Builders¶

In the previous example, you have created the user-interface for the application by calling the create functions for each widget and placing it where you wanted it. However, this can be quite burdensome for more complex applications. Many programming environments, including GNOME, have programs called GUI builders that can be used to automatically create your GUI for you. You just have to write the code for the signal handlers and for initializing your program. The main GUI builder for GNOME applications is called GLADE. GLADE ships with most Linux distributions.

There are GUI builders for most programming environments. Borland has a range of tools that will build GUIs quickly and easily on Linux and Win32 systems. The KDE environment has a tool called QT Designer which helps you automatically develop the GUI for their system.

There is a broad range of choices for developing graphical applications, but hopefully this appendix gave you a taste of what GUI programming is like.
