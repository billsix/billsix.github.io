Computation at Compile-Time

Computation at Compile-Time is both the documentation for and the implementation of libbug, a library for Gambit Scheme. Libbug introduces a truly revolutionary way to test code. Tests are collocated with the procedures under test, typically directly after a procedure's definition.



It's "programmer-tested, rapper-approved".



But wait. Doesn't this mean that the test will be in the executable, and that it will run every time the program runs? That would be unacceptable. Fortunately, the answer to both is no, because the tests execute during compile-time, and they are not compiled into the executable.

How does that work? It works because every procedure is loaded into the compiler as code to be interpreted (by evaling during macroexpansion), and afterword it is treated as code to be compiled. Thus, libbug treats the compiler as an interpreter, both of which use the same language and have the same libraries (in stark contrast to C++'s template metaprogramming).
The definition of "unit-test" uses the same trick, but if any of the tests fail, the compiler exits in error, just like a type-error in a statically-typed language. If all of the tests pass, the "unit-test" evaluates to the symbol "noop", which the Gambit compiler eliminates as dead code.
