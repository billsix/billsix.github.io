Add Translate Method to Vertex - Demo 05¶

Purpose¶

Restructure the code towards future knowledge, i.e. the model view projection pipeline.

Transforming vertices, such as translating, is the core concept of computer graphics.

Translation¶

Rather than incrementing y values before calling “glVertex”, instead call “translate” on the vertex, and call “glVertex2f” on the translated vertex.

Returning a new instance will be very useful for method chaining. I would not use this code for production code, because of efficincy, but the use of method chaining will be towards furthering your understanding of a sequence of transformations, and the order in which you probably want to read them.

 class Vertex:
     def __init__(self, x, y):
         self.x = x
         self.y = y

     def __repr__(self):
         return f"Vertex(x={repr(self.x)},y={repr(self.y)})"


     def translate(self, tx, ty):
         return Vertex(x=self.x + tx, y=self.y + ty)
